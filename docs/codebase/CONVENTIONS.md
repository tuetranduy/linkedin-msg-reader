## File Naming

| Layer              | Convention           | Examples                                          |
|--------------------|----------------------|---------------------------------------------------|
| React components   | PascalCase + `.tsx`  | `MessageBubble.tsx`, `ConversationList.tsx`       |
| Context files      | PascalCase + `.tsx`  | `AuthContext.tsx`, `MessageContext.tsx`            |
| Hooks              | camelCase + `.ts`    | `useMediaQuery.ts`                                |
| Utilities          | camelCase + `.ts`    | `csvParser.ts`, `utils.ts`                        |
| Type files         | camelCase + `.ts`    | `message.ts`, `room.ts`                           |
| Backend routes     | camelCase + `.mongo.ts` | `conversations.mongo.ts`, `auth.mongo.ts`      |
| Backend middleware | camelCase + `.mongo.ts` | `auth.mongo.ts`                                |
| Backend db         | camelCase + `.ts`    | `mongodb.ts`, `database.ts`                       |

> Evidence: `src/components/`, `src/context/`, `server/src/routes/`

## Component Exports

- **Named exports only** — no default exports for React components
- Example: `export function MessageBubble(...)` in `MessageBubble.tsx`
- Evidence: all files in `src/components/`

## Function & Variable Naming

- `camelCase` for all variables, functions, methods
- `PascalCase` for React components, TypeScript interfaces, type aliases
- Socket event handlers: `socket.on('room:create', ...)` — kebab with colons for events
- MongoDB field names: `snake_case` in DB (e.g., `conversation_id`, `is_visible`, `last_message_date`)
- JavaScript/TypeScript layer: `camelCase` (e.g., `conversationId`, `isVisible`, `lastMessageDate`)
- Evidence: `server/src/routes/conversations.mongo.ts`, `src/types/message.ts`

## Import Style

- Backend ESM: all relative imports use `.js` extension (ESM Node.js / `"module": "NodeNext"`)
  ```ts
  import { getDB, ObjectId } from '../db/mongodb.js'
  import { authenticateToken } from '../middleware/auth.mongo.js'
  ```
- Frontend: no extension on relative imports (bundler mode)
  ```ts
  import { apiClient } from '../api/client'
  import { useAuth } from './context/AuthContext'
  ```
- Path alias `@/` maps to `src/`
- No barrel `index.ts` exports observed — direct file imports used throughout

## Error Handling

### Backend
- Route handlers return structured JSON error objects:
  ```ts
  res.status(404).json({ error: 'Resource not found' })
  res.status(401).json({ error: 'Authentication required' })
  res.status(500).json({ error: 'Internal server error' })
  ```
- Global error handler in `server/src/index.ts` catches unhandled errors
- Socket handlers wrap all logic in try/catch, emit `room:error` or call `callback({ error: '...' })`

### Frontend
- `apiClient<T>()` throws `ApiError` (extends `Error`) with `.status` on non-2xx responses
- Components catch errors in async handlers and store in local `error` state
- Error display: `bg-red-50 text-red-900` styled `<div>` block

## Logging

- Backend uses `console.log` / `console.error` (no structured logging library)
- Socket events log user actions: `console.log(\`User connected: ${user.username}\`)`
- No log levels, no correlation IDs, no log aggregation tool
- Evidence: `server/src/socket/handlers.ts`, `server/src/index.ts`

## React Patterns

### Context Consumption
Always use the exported custom hook:
```ts
const { user, isAuthenticated } = useAuth()     // never useContext(AuthContext) directly
const { conversations, selectedConversation } = useMessages()
const { currentRoom, isInRoom } = useRoom()
```

### API Calls
```ts
import { apiClient, ApiError } from '../api/client'
const data = await apiClient<MyType>('/endpoint', { method: 'POST', body: JSON.stringify(payload) })
```

### Auth Token
- Stored at `localStorage.getItem('auth_token')` — never read directly in components; use `useAuth()`
- Set/cleared only inside `AuthContext`

## Styling Conventions

- Tailwind CSS utility-first; no CSS modules or styled-components
- Mobile-first responsive breakpoints: `sm:`, `md:`
- Card pattern: `border rounded p-4` with `truncate` for overflow
- Loading state: `disabled` button + `Loader2` spinner icon from `lucide-react`
- Dark mode: toggled via `document.documentElement.classList.toggle('dark')`, persisted in `localStorage.darkMode`
- Icon library: `lucide-react` exclusively

## Backend Route Pattern

```ts
const router = Router()
router.use(authenticateToken)     // auth applied at router level

router.get('/', async (req: AuthRequest, res: Response): Promise<void> => {
  // handler body
})

export default router
```
